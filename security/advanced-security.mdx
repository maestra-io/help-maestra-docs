The **Advanced Security** provides extra security features for large businesses with special security requirements.

## **What is included**

## Masking personal data for specific users

This feature allows you to hide personal data from selected staff members.

![](/images/imported/Advanced-security.png)

Personal data includes: full name, email, and phone number.
​ 

How to set up:

1. Go to the **Settings tab → Security → Security Policy** and enable **Hide customer personal data**.

![](/images/imported/Advanced-2Bsecurity-2B-281-29.png)

2. Control data visibility by granting or removing the **View and edit personal data** permission for the relevant staff or security groups.

![](/images/imported/Advanced-security--282-29.png)

For more flexible control, assign the ***Unmask personal data*** permission to a user. 

This allows them to view customer data on the customer’s page.

![](/images/imported/Advanced-security--283-29.png)

Every use of this feature is recorded in the event log.

![](/images/imported/Advanced-security--285-29.png)

<Note>

Note: **Export restrictions** can be set without Advanced Security. Simply remove the **Export personal data** permission from staff or security groups.
</Note>

## **Restricted access by IP**

This feature allows staff to log in to the platform only from specific IP addresses.

To enable it, provide your Mindbox manager with a list of trusted IP addresses.

Once added to the platform’s web server, access to the project will be allowed only from these addresses.

## **Exporting events to SIEM (Security Information and Event Management)**

This allows you to **regularly export logs** from the event log, which records:

- Project logins with IP addresses
- Creation, editing, and deletion of campaigns, segments, customers, and other entities
- Export and import bulk tasks
- Changes to staff and permissions, and more.

![](/images/imported/Advanced-security--286-29.png)

## **SSO — Single Sign-On**

This feature lets you manage staff access centrally.

Learn more about SSO [here](/security/single-sign-on-sso).

## **How to enable** 

The Advanced Security is enabled by default and does not require an extra fee.
